HTML gir strukturen. CSS gir utseendet. Med CSS kan du endre farger, fonter, størrelser, avstander og plassering   alt som gjør en nettside fin og brukervennlig.

## Hva er CSS?

CSS (Cascading Style Sheets) er et språk som bestemmer hvordan HTML-elementer skal se ut. En CSS-regel ser slik ut:

```css
h1 {
  color: blue;
  font-size: 2rem;
}
```

Dette betyr: "Alle `<h1>`-elementer skal ha blå tekst og skriftstørrelse 2rem."

CSS-regler består av:
- **Selektor**: hvilket element som styles (`h1`, `p`, `.klasse`, `#id`)
- **Egenskaper**: hva som styles (`color`, `font-size`, `margin`)
- **Verdier**: hvordan det styles (`blue`, `2rem`, `20px`)

## Hvordan legges CSS til?

Det er tre måter å bruke CSS på. Den beste måten er en egen CSS-fil:

```html
<link rel="stylesheet" href="style.css">
```

Denne linken settes inn i `<head>`-delen av HTML-en. Da kan du ha all styling i én fil, og HTML-en forblir ryddig.

## De vanligste CSS-egenskapene

### Farger

```css
body {
  background-color: #f5f5f5;  /* Lys grå bakgrunn */
  color: #333;                 /* Mørk grå tekst */
}

h1 {
  color: #1a5276;              /* Mørk blå overskrift */
}
```

Farger angis oftest som hex-koder (`#RRGGBB`). Du kan også bruke navn (`red`, `blue`) eller `rgb(255, 0, 0)`.

### Skrift

```css
body {
  font-family: Arial, Helvetica, sans-serif;
  font-size: 16px;
  line-height: 1.6;      /* Linjeavstand */
  text-align: center;     /* Venstre, høyre, senter */
}
```

### Avstand

```css
.card {
  margin: 20px;           /* Avstand UTENFOR elementet */
  padding: 15px;          /* Avstand INNENI elementet */
  border: 1px solid #ddd; /* Ramme rundt */
  border-radius: 8px;     /* Runde hjørner */
}
```

Boksen rundt hvert element består av (innerst til ytterst): **innhold → padding → border → margin**.

### Størrelse og synlighet

```css
.bilde {
  width: 100%;           /* Bredde i prosent av forelderen */
  max-width: 600px;       /* Men ikke større enn 600px */
  height: auto;           /* Behold proporsjoner */
  display: none;          /* Skjult (fjernes fra sidestrømmen) */
  opacity: 0.5;           /* Gjennomsiktighet (0=usynlig, 1=synlig) */
}
```

## Layout med Flexbox

Flexbox er en moderne måte å arrangere elementer på. Her er det viktigste:

```css
.container {
  display: flex;           /* Gjør dette til en flexbeholder */
  justify-content: center; /* Sentrer horisontalt */
  align-items: center;     /* Sentrer vertikalt */
  gap: 20px;               /* Avstand mellom barna */
}
```

Du trenger ikke å kunne Flexbox utenat   du beskriver hva du vil, og KI-en skriver CSS-en. Men å vite at `display: flex` er magien som gjør layout mulig, hjelper deg å forstå hva KI-en har laget.

## Responsivt design (mobilvennlig)

En nettside må fungere på både store skjermer og mobiltelefoner. Dette kalles responsivt design. Nøkkelverktøyet er `@media`-regler:

```css
/* Standardstil for store skjermer */
.grid {
  display: grid;
  grid-template-columns: 1fr 1fr 1fr; /* 3 kolonner */
}

/* Smalere skjermer (tablets) */
@media (max-width: 768px) {
  .grid {
    grid-template-columns: 1fr 1fr;   /* 2 kolonner */
  }
}

/* Mobiltelefoner */
@media (max-width: 480px) {
  .grid {
    grid-template-columns: 1fr;       /* 1 kolonne */
  }
}
```

Du trenger ikke skrive dette selv. Bare be OpenCode: *"Gjør nettsiden min mobilvennlig"* eller *"Legg til responsive media queries"*.

:::tip[Test på mobil]
Etter at KI-en har gjort siden mobilvennlig, åpne nettleserens utviklerverktøy (høyreklikk → "Inspiser" → trykk på mobil-ikonet) for å se hvordan siden ser ut på en telefon.
:::

## Fallgruver i CSS

### Fallgruve: For kompleks CSS
KI-en har en tendens til å generere veldig kompleks CSS med mange regler når en enkel løsning ville fungert. Hvis du ser veldig mye CSS for en enkel endring, spør: *"Kan du forenkle CSS-en?"*

### Fallgruve: !important som krykke
Hvis KI-en setter `!important` på CSS-regler, er det ofte et tegn på at noe er rotete. Be om å få ryddet opp i stedet: *"Fjern !important og fiks selektorene i stedet."*

### Fallgruve: Ikke mobilvennlig
Som standard lager KI-en ofte for store skjermer. Hvis du ikke spesifikt ber om mobilvennlig design, kan siden se ødelagt ut på telefon. Legg alltid til i prompten: *"Gjør den mobilvennlig"*.
