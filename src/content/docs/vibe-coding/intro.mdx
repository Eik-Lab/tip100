Velkommen til Vibe Coding   en helt ny måte å lage nettsider og digitale produkter på. I stedet for å skrive hver linje kode for hånd, samarbeider du med en kunstig intelligens (KI) som skriver koden for deg, basert på instruksjonene du gir.

## Hva er Vibe Coding?

Vibe Coding er et begrep som beskriver utvikling hvor du lar KI-assistenter som OpenCode generere mesteparten av koden. Du forklarer hva du vil ha på vanlig språk (norsk eller engelsk), og KI-en skriver HTML, CSS og JavaScript for deg.

Tenk på det som å ha en senior utvikler som sitter ved siden av deg og gjør alt det tunge skrivearbeidet   mens du bestemmer hva som skal lages og kvalitetssikrer resultatet.

**Hva du kan bygge med Vibe Coding:**
- Landingssider og produktnettsider
- Portefølje- og visittkortsider
- Enkle verktøy og kalkulatorer
- Prototyper for nye idéer
- Interaktive quizzer og gallerier

Hva du **ikke** bør bygge med KI alene: bank- og betalingssystemer, medisinske applikasjoner, innloggingssystemer med brukerdata, eller annen kode som håndterer sensitiv informasjon. KI kan gjøre feil, og du er ansvarlig for sluttproduktet.

## Hvordan fungerer en KI-assistent?

KI-assistenter som OpenCode er trent på enorme mengder kode fra internett. Når du skriver en prompt (beskjeden du gir KI-en), analyserer den hva du spør om og genererer kode basert på mønstre den har lært.

Dette betyr at KI-en:
- Er veldig god på vanlige, velkjente oppgaver
- Kan foreslå flere måter å løse samme problem
- Av og til tar feil, og er veldig overbevisende når den tar feil
- Kan finne på funksjoner eller biblioteker som ikke finnes (hallusinering)

**Dette er den viktigste beskjeden i hele kurset: KI tar feil. Ofte. Den høres trygg og overbevisende ut, men den tar feil. Du må alltid sjekke koden den genererer.**

## Tankesettet for Vibe Coding

For å lykkes med Vibe Coding trenger du riktig tankesett:

1. **Iterer, ikke perfeksjoner**   Første versjon er sjelden den beste. Planlegg 5-10 oppfølgingsprompter per sideelement.

2. **Eksperimenter**   Prøv forskjellige formuleringer. Hvis en prompt ikke fungerer, omformuler den.

3. **Feil er læring**   Når koden ikke virker, er det en mulighet til å forstå hvordan ting henger sammen. Ikke bare be KI-en fikse det   spør hvorfor det ble feil.

4. **Verifiser alt**   Test hver funksjon. Sjekk koden. Åpne nettleseren og se at alt fungerer.

5. **Du bestemmer**   KI-en er verktøyet, du er arkitekten. Du bestemmer retning, struktur og kvalitetsnivå.

## Hva du vil lære i dette kurset

Kurset tar deg fra null til en publisert nettside, steg for steg:

| Modul | Tema |
|---|---|
| 1 | Hva er Vibe Coding? (denne siden) |
| 2 | Slik fungerer nettsider |
| 3 | Installasjon av verktøy |
| 4 | Din første nettside |
| 5 | Prompting som en proff |
| 6 | Forstå HTML-koden |
| 7 | Design og styling |
| 8 | Gjør nettsiden interaktiv |
| 9 | Bygg et prosjekt steg for steg |
| 10 | Feilsøking med AI |
| 11 | Publisering |
| 12 | Sikkerhet og personvern |
| 13 | Vanlige fallgruver |
| 14 | Etikk og ansvarlig bruk |

På slutten av kurset vil du ha bygget og publisert din egen nettside   uten å kunne kode fra før.
